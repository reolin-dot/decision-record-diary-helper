# 决策成长日记 Web App 版本规划

日期：2026-06-29  
适用范围：当前 `web-app` 版本后续迭代  
产品原则：保留种花和治愈感，但把花园降为外壳；核心价值来自决策记录、行动推进、复盘和自我洞察。

---

## 0. 当前版本状态

当前 Web 版已经完成从小程序到 Vite + React 的主要迁移，核心闭环基本可用。

已完成能力：

- 花园首页：决策状态展示、最近决策、待浇水提醒、基础统计。
- 记录流程：4 步记录决策，支持草稿、正式保存、复盘时间设置。
- 决策详情：查看单条决策完整信息，支持开始行动、进入复盘。
- 浇水/复盘：支持主动复盘和到期复盘，区分当下复盘与结果复盘。
- 决策风格测试：已接入快速版、标准版、深入版题库。
- 锦囊：已按 5 类困境提供不同思考框架，并支持保存到决策记录。
- 本地数据：通过 StorageAdapter 和 localStorage 保存，支持导出/导入 JSON 备份。
- 个人页：展示统计、徽章、风格结果、数据保存说明和功能入口。

当前边界：

- 2.0 正式版不做账号登录和云同步，先以本地浏览器保存 + JSON 导出/导入备份保障数据安全。
- 2.1 之后重新纳入账号登录和云同步，但作为后续运营化能力，不阻塞 2.0 发布。
- 暂无浏览器通知或系统级提醒。
- 月度报告和成长档案入口已在 V2.0-alpha.1 迁入；分享卡、主题花园仍只作为旧小程序参考。
- 目前更像“可用 MVP”，还不是“可长期运营的产品”。

## 1. V1.1 稳定体验版

目标：把当前可用版本打磨成稳定、舒服、可连续体验的第一版。

建议周期：3 到 5 天。

### 1.1 必做任务

- 全流程走查：新用户进入、记录决策、保存、查看详情、开始行动、主动浇水、结果复盘、导出备份。
- 文案统一：全站统一使用“浇水/复盘”“种子/发芽/长叶/初开/盛开”的表达。
- 异常状态补齐：空数据、找不到决策、导入失败、存储失败、无可浇水决策。
- 本地保存提示前置：在首次进入或个人页明确告知“数据保存在本机浏览器”。
- 移动端适配检查：重点看 iPhone 窄屏、安卓长屏、桌面窄容器。
- 构建检查：每次交付前必须跑 `npm run build`。

### 1.2 可选增强

- 给花朵阶段切换增加轻量动效。
- 给“完成浇水”增加更有仪式感的反馈卡片。
- 花园首页增加“继续完成草稿”的明确入口。
- 导出文件名改为带日期，例如 `decision-diary-backup-2026-06-29.json`。

### 1.3 验收标准

- 用户不用看说明，也能完成一条完整决策闭环。
- 关键页面无乱码、无空白、无明显按钮失效。
- 刷新页面后数据仍在。
- 导出后可重新导入恢复。
- `npm run build` 通过。

## 2. V1.2 留存与复盘增强版

目标：让用户更愿意回来复盘，而不是只记录一次就走。

建议周期：1 到 2 周。

### 2.1 站内提醒中心

先不做浏览器通知，先做应用内提醒。

功能建议：

- 新增“提醒中心”或在浇水页顶部展示提醒分组。
- 到期复盘：`reviewDate <= today`。
- 行动提醒：记录后 3 天仍未标记开始行动。
- 结果跟进：初开后 7 天仍未完成结果复盘。
- 支持“稍后提醒”“今天不提醒”“直接去复盘”。

文案原则：

- 不催促。
- 不评价。
- 强调“回来看看”，不是“必须完成”。

### 2.2 成长片段

把复盘中的经验沉淀出来，形成用户愿意回看的资产。

功能建议：

- 从 `wateringHistory.lesson` 中提取成长片段。
- 新增“成长片段”列表页。
- 支持按决策、日期、关键词查看。
- 花园首页展示最近 1 到 3 条片段。

### 2.3 复盘质量提升

现在复盘能完成，但还可以更像“反思工具”。

功能建议：

- 当下复盘问题聚焦“新信息、推进状态、下一步”。
- 结果复盘问题聚焦“事实、判断验证、下次保留/调整”。
- 完成复盘后生成一句温柔总结。
- 对“结果不理想”的情况避免残忍反馈，强调信息增量。

### 2.4 验收标准

- 用户能清楚知道哪些决策需要回来看看。
- 完成复盘后能留下可回看的成长片段。
- 花园不只是展示记录数量，而是展示成长进度。

## 3. V1.3 决策辅助增强版

目标：让产品不只是“记录工具”，还可以帮助用户想清楚。

建议周期：1 到 2 周。

### 3.1 锦囊深化

当前已有 5 类锦囊，可以继续做得更有“产品感”。

功能建议：

- 每类锦囊结果页增加更明确的“你现在可以怎么做”。
- 锦囊结果保存到决策后，记录页标记来源为“锦囊生成”。
- 在决策详情中显示“来自哪个锦囊”。
- 支持从决策详情再次打开对应锦囊。

### 3.2 决策模板

为常见场景提供轻模板，降低记录门槛。

模板建议：

- 职业选择：换工作、接 offer、转方向。
- 学习成长：是否报课、是否坚持某计划。
- 消费决策：是否购买大件、是否订阅服务。
- 关系沟通：是否表达、是否结束、是否推进。
- 时间分配：要不要接一个新任务。

注意：模板不要太死板，应该是“提示用户想一想”，不是“替用户填表”。

### 3.3 风格测试参与推荐

让风格测试结果真正进入产品体验。

功能建议：

- 理性分析型：提醒设置时间上限，避免过度分析。
- 直觉行动型：提醒补充关键风险，避免冲动。
- 情绪敏感型：优先推荐情绪降噪锦囊。
- 信息收集型：推荐最低成本验证，而不是无限调研。

### 3.4 验收标准

- 用户遇到纠结时能先用锦囊，再自然转成决策记录。
- 决策风格测试不再是孤立功能，而能影响提示和推荐。
- 记录流程比现在更轻，不会让用户一开始就觉得负担重。

## 4. V1.4 本地数据安全版

目标：不做账号和云端，也让用户敢长期记录。

建议周期：1 周。

### 4.1 本地备份增强

功能建议：

- 导出完整备份时带 `schemaVersion`。
- 导入前展示备份中的决策数量和导出时间。
- 支持“合并导入”和“覆盖导入”两种模式。
- 支持导入失败时给出更明确原因。

### 4.2 本地恢复体验

功能建议：

- 导出页明确显示“数据保存在当前浏览器”。
- 提供快速备份、完整备份、导入预览和低风险修复。
- 导入或修复前自动提示先下载当前备份。
- 给出“换设备/清浏览器数据会丢失本地记录”的边界说明。

### 4.4 验收标准

- 用户清楚知道当前数据在哪里。
- 用户能自己备份和恢复。
- 换设备或清缓存这类风险有明确提醒和备份路径。

## 5. V2.0 成长系统版

目标：从单次决策工具升级为长期成长档案。

建议周期：3 到 6 周。

### 5.1 月度成长报告

可以参考 `reference/legacy-miniprogram-v2/pages/monthly-report`。

报告内容建议：

- 本月记录了多少次决策。
- 完成了多少次浇水/复盘。
- 最常出现的决策主题。
- 最常见的决策风格倾向。
- 本月最有价值的 3 条成长片段。
- 给下个月的一句提醒。

### 5.2 决策模式洞察

功能建议：

- 统计用户常见决策类型。
- 统计拖延、反复纠结、情绪影响、信息不足等模式。
- 用非常温和的方式提示：“你最近可能更容易卡在信息收集阶段。”
- 给出下一次可尝试的小策略。

### 5.3 分享卡

可以参考 `reference/legacy-miniprogram-v2/pages/share-card`。

分享内容建议：

- 决策风格测试结果卡。
- 本月成长报告卡。
- 花园阶段卡。
- 单条成长片段卡。

注意：分享不能泄露具体隐私。默认只分享抽象结果，不分享决策标题和详情。

### 5.4 主题花园

可以参考 `reference/legacy-miniprogram-v2/pages/theme-garden`。

功能建议：

- 不同决策类型对应不同植物或区域。
- 职业、学习、关系、消费、健康、生活分别形成小花圃。
- 用户看到的不是“我有很多记录”，而是“我在哪些人生主题上持续成长”。

## 6. V2.1 账号与身份版

目标：在不打断本地使用的前提下，给长期用户一个可迁移、可保护的身份入口。

建议周期：1 到 2 周。

### 6.1 登录策略

功能建议：

- 保留游客模式，首次进入仍可直接记录。
- 个人页和数据导出页新增登录入口，不放在首页前置拦截。
- 登录文案强调“保护记录”和“跨设备恢复”，不要强调“注册账号”。
- 登录后先绑定当前本地数据，不自动覆盖。

### 6.2 账号范围

功能建议：

- 支持邮箱或第三方登录，先选一种最容易落地的方式。
- 账号只保存身份、基础设置和数据归属，不做社交关系。
- 登录失败时允许继续使用本地模式。
- 明确退出登录后，本地已有数据仍保留。

### 6.3 验收标准

- 未登录用户仍能完成完整决策闭环。
- 登录用户能看到账号状态和数据保护说明。
- 登录不会造成本地数据丢失或重复覆盖。

---

## 7. V2.2 云备份与同步版

目标：解决换设备、清浏览器数据和长期保存的问题。

建议周期：2 到 4 周。

### 7.1 云备份优先

先做云备份，再做实时多端同步。

功能建议：

- 登录后支持手动上传当前本地备份。
- 支持从云端恢复最近一次备份。
- 云端备份展示时间、记录数量和版本号。
- 恢复前展示影响范围，并自动下载当前本地备份。

### 7.2 增量同步

功能建议：

- 关键操作后同步：新增决策、完成复盘、保存 AI 洞察。
- 使用 `updatedAt` 判断新旧，先采用最后写入优先。
- 冲突时优先不自动合并，先提示用户选择保留本地或云端。
- 失败时保留本地数据，并提示稍后重试。

### 7.3 验收标准

- 用户换设备后能通过账号恢复数据。
- 网络失败不会影响本地记录。
- 云端恢复前用户能看清楚影响范围。

---

## 8. 推荐优先级

如果只能按顺序做，建议这样排：

1. V1.1 稳定体验版：先保证当前产品像一个完整作品。
2. V1.2 留存与复盘增强版：解决用户为什么回来。
3. V1.3 决策辅助增强版：解决用户为什么需要它。
4. V1.4 本地数据安全版：解决用户敢不敢长期记录。
5. V2.0 成长系统版：解决产品能不能形成长期价值。
6. V2.1 账号与身份版：解决长期用户的数据归属。
7. V2.2 云备份与同步版：解决跨设备和数据恢复。

## 9. 给后续模型的执行提示

执行任何版本任务前，先阅读：

- `README.md`
- `docs/version-roadmap-web-app-2026-06-29.md`
- `docs/question-bank/README.md`
- `docs/plans/小程序转Web应用_23bf9e89.md`

每次改动后必须做：

```powershell
Set-Location 'E:\决策成长日记\web-app'
npm run build
```

如果涉及体验流，至少手动跑一遍：

- 记录一条决策。
- 从花园进入详情。
- 标记开始行动。
- 主动浇水。
- 导出数据。
- 刷新页面确认数据还在。
