# 决策成长日记 Web App 版本更新记录

> 记录口径：从 V1.4 之后开始，按“本地可信 -> 复盘留存 -> 长期成长档案 -> 主题花园”的顺序整理。2.0 正式版仍以本地保存、JSON 备份和导入恢复为主；账号登录和云同步重新纳入 2.1/2.2 后续规划。

## V1.6 数据安全与本地闭环增强版 - 2026-07-06

### 版本目标

补齐“用户敢不敢长期记录”的信任问题，把本地保存、备份、导入、修复和外部 AI 分析串成闭环。

### 更新内容

- 新增本地数据健康检查，帮助用户知道当前数据是否完整可用。
- 新增备份新鲜度提示，提醒用户定期保存本地备份。
- 新增快速备份和完整本地备份，降低长期记录的数据丢失风险。
- 新增导入前影响预览，让用户知道会新增、合并或影响多少数据。
- 导入或修复前自动下载当前备份，避免误操作造成不可恢复损失。
- 新增低风险数据修复，只处理空数组、默认状态、布尔字段等格式问题。
- 新增 DeepSeek 分析包导出，方便用户把结构化数据交给外部 AI 分析。
- 新增 AI 成长洞察保存，把外部 AI 返回内容沉淀回产品内。
- 新增个性化复盘问题、复盘状态选项、跟进提示和首页决策模式洞察。

### 验收重点

- 用户知道数据保存在当前浏览器。
- 用户可以导出备份、导入恢复，并在导入前看到影响。
- 数据修复只处理低风险问题，不自动改写高风险结构。

## V1.5 本地备份与迁移安全网 - 2026-07-06

### 版本目标

在不做账号和云同步的前提下，先建立“可导出、可恢复、可说明”的数据安全策略。

### 更新内容

- 明确 Web 版采用 localStorage 作为本地保存方式。
- 明确通过 Storage Adapter 访问数据，为导出、导入、迁移和异常处理留统一入口。
- 规划 JSON 备份文件结构，包含决策记录、复盘历史、成长片段和 AI 洞察。
- 明确备份文件包含隐私内容，需要提示用户妥善保存。
- 明确换设备、换域名或清理浏览器数据前，需要先导出完整备份。
- 将云同步、账号登录从当前路线图移除，避免 MVP 阶段过早引入后端复杂度。

### 验收重点

- 产品说明里能清楚解释“数据在哪里”。
- 后续数据安全能力围绕本地备份恢复展开，而不是账号和云端。
- 迁移计划与版本路线图口径一致。

## V1.4 本地数据安全方向确定版 - 2026-07-06

### 版本目标

把后续方向从“先接云同步”收敛为“先做好本地可信”，控制项目范围。

### 更新内容

- 确定不规划账号登录体系，降低产品和技术复杂度。
- 确定不规划云同步，优先保证单端本地使用体验。
- 将数据安全目标拆成三件事：知道数据在哪里、能导出备份、能导入恢复。
- 明确导出完整备份需要带 `schemaVersion`，方便后续迁移和兼容。
- 明确导入前需要展示备份内容和影响范围，避免用户误覆盖数据。
- 明确异常数据只做低风险修复，不自动处理结构性错误。

### 验收重点

- 版本路线图不再把账号、云同步作为后续必做项。
- 本地数据安全成为 V1.6 之后的基础能力。
- 后续所有成长系统能力都建立在本地闭环上。
