## 一、数据分析目标

### 1.1 产品目标与数据目标的对应关系

| 产品目标 | 数据目标 | 衡量指标 |
|---------|---------|---------|
| 帮助用户养成记录决策的习惯 | 追踪用户活跃度与留存 | DAU/MAU、次日留存、7日留存 |
| 提升决策复盘完成率 | 监控复盘行为转化 | 复盘完成率、复盘提醒打开率 |
| 提供有价值的决策辅助 | 评估锦囊功能使用效果 | 锦囊使用率、分析结果查看率 |
| 增强用户成长可视化体验 | 追踪花园页面互动 | 页面停留时长、种花/浇水点击率 |
| 建立长期用户粘性 | 分析用户生命周期价值 | LTV、连续记录天数分布 |

### 1.2 数据驱动决策的意义

1. **产品迭代方向**: 通过用户行为数据识别高频/低频功能，优化产品路线图
2. **用户体验优化**: 通过漏斗分析定位流失节点，降低用户操作阻力
3. **个性化推荐**: 基于用户决策类型偏好，智能推荐相关锦囊模板
4. **留存提升**: 通过分群分析识别高价值用户特征，制定精准留存策略
5. **商业化铺垫**: 建立用户行为画像，为未来会员增值服务奠定数据基础

## 三、漏斗分析设计

### 3.1 决策记录完成漏斗

```
启动小程序 → 进入花园页 → 点击"种花" → 完成Step 1 → 完成Step 2 → 完成Step 3 → 完成Step 4 → 记录成功
```

**漏斗步骤与埋点对应**:

| 漏斗步骤 | 对应埋点事件 | 转化率计算基准 |
|---------|-------------|--------------|
| 启动小程序 | `app_launch` | 100% (基准) |
| 进入花园页 | `garden_view` | `garden_view / app_launch` |
| 点击"种花" | `garden_plant_click` | `garden_plant_click / garden_view` |
| 完成Step 1 | `record_step_view` (step=1) + `record_title_complete` | `title_complete / plant_click` |
| 完成Step 2 | `record_option_add` (至少添加1个选项) | `option_add / title_complete` |
| 完成Step 3 | `record_solution_select` | `solution_select / option_add` |
| 完成Step 4 | `record_review_time_set` | `review_time_set / solution_select` |
| 记录成功 | `record_complete` | `record_complete / review_time_set` |

**关键流失点分析**:
- **Step 1 → Step 2**: 标题填写后放弃，可能是决策思路不清晰
- **Step 2 → Step 3**: 选项添加后放弃，可能是选项过多导致选择困难
- **Step 3 → Step 4**: 方案选择后放弃，可能是复盘时间设置阻力
- **整体完成率目标**: 初期设定为 40%，优化后目标 60%

### 3.2 复盘完成漏斗

```
收到复盘提醒 → 打开提醒 → 进入复盘页 → 填写复盘内容 → 提交复盘
```

**漏斗步骤与埋点对应**:

| 漏斗步骤 | 对应埋点事件 | 转化率计算基准 |
|---------|-------------|--------------|
| 收到复盘提醒 | `notification_receive` | 100% (基准) |
| 打开提醒 | `notification_receive` (is_clicked=true) | `clicked / received` |
| 进入复盘页 | `review_view` | `review_view / clicked` |
| 填写复盘内容 | `review_complete` (结果填写中) | 通过页面停留时长判断 |
| 提交复盘 | `review_complete` | `complete / review_view` |

**关键流失点分析**:
- **提醒打开率低**: 推送通知权限未开启或提醒时间不合适
- **进入复盘页后放弃**: 复盘表单设计复杂度过高
- **复盘完成率目标**: 初期设定为 30%，优化后目标 50%

### 3.3 锦囊使用漏斗

```
进入锦囊页 → 选择困境类型 → 填写分析框架 → 查看分析结果 → 应用到决策
```

**漏斗步骤与埋点对应**:

| 漏斗步骤 | 对应埋点事件 | 转化率计算基准 |
|---------|-------------|--------------|
| 进入锦囊页 | `coach_view` | 100% (基准) |
| 选择困境类型 | `coach_type_select` | `type_select / coach_view` |
| 填写分析框架 | 框架填写进度（需补充埋点） | 框架完成度追踪 |
| 查看分析结果 | `coach_analysis_view` | `analysis_view / type_select` |
| 应用到决策 | `coach_apply_click` | `apply_click / analysis_view` |

**锦囊功能核心价值**: 引导用户结构化思考，提升决策质量

## 四、用户分群策略

### 4.1 按用户生命周期分群

| 用户群体 | 定义 | 运营策略 |
|---------|------|---------|
| 新用户 | 注册后7天内 | 引导完成首次决策记录，推送锦囊模板 |
| 活跃用户 | 近7天有登录 | 鼓励连续记录，推送复盘提醒 |
| 沉默用户 | 7-30天未登录 | 推送成长报告，唤醒决策回顾 |
| 流失用户 | 30天以上未登录 | 分析流失原因，定向召回 |

### 4.2 按使用频率分群

| 用户群体 | 定义 | 特征分析 |
|---------|------|---------|
| 高频用户 | 每周记录≥3次决策 | 对产品核心价值认可，可能是KOL种子用户 |
| 中频用户 | 每周记录1-2次决策 | 产品主力用户，需持续激励 |
| 低频用户 | 每周记录<1次决策 | 需降低使用门槛，提升唤醒频率 |
| 试用用户 | 仅记录1次后不再使用 | 需分析首次体验阻力点 |

### 4.3 按行为特征分群

| 用户群体 | 定义 | 分析维度 |
|---------|------|---------|
| 复盘完成者 | 复盘完成率≥50% | 成长意识强，对可视化反馈敏感 |
| 复盘跳过者 | 复盘完成率<50% | 需简化复盘流程，或调整提醒策略 |
| 锦囊重度用户 | 使用锦囊≥3次/周 | 对决策辅助需求强，可推荐高级模板 |
| 花园互动用户 | 频繁点击浇水/查看花朵 | 对成长可视化认同感强 |

### 4.4 分群分析应用场景

1. **新用户 vs 老用户**:
   - 对比首次决策记录完成率
   - 分析功能使用路径差异
   - 制定差异化引导策略

2. **高频用户 vs 低频用户**:
   - 分析使用时段分布
   - 对比决策类型偏好
   - 识别高频用户特征，复制给低频用户

3. **完成复盘用户 vs 跳过复盘用户**:
   - 分析复盘提醒打开率差异
   - 对比决策复杂度
   - 优化复盘表单设计

### 5.4 数据看板技术实现建议

#### 5.4.1 看板工具选择

1. **快速启动方案**: 微信小程序数据助手 + Excel/Google Sheets
   - 优点: 免费、快速上手
   - 缺点: 功能有限、无法自定义

2. **专业方案**: 神策数据 / GrowingIO / 诸葛IO
   - 优点: 功能强大、可视化丰富、支持漏斗/留存等高级分析
   - 缺点: 需要付费、接入成本较高

3. **自建方案**: Metabase / Superset + 自建数据仓库
   - 优点: 完全自主可控、成本可控
   - 缺点: 开发成本高、维护成本高

**推荐路径**: 
- Phase 1 (0-3个月): 微信小程序数据助手 + 自建简单看板
- Phase 2 (3-6个月): 接入神策数据基础版
- Phase 3 (6个月+): 根据需求升级或自建

#### 5.4.2 看板更新频率

| 看板类型 | 更新频率 | 受众 |
|---------|---------|------|
| 实时概览 | 每小时刷新 | 产品负责人、运营 |
| 核心指标 | 每日更新 | 产品团队 |

## 七、数据安全与合规

### 7.1 数据匿名化
- 用户ID使用UUID,不与微信OpenID直接关联
- 敏感决策内容本地加密存储(后端接入后)

### 7.2 用户授权
- 明确告知用户数据收集范围与用途
- 提供数据删除选项(符合GDPR/个人信息保护法)

### 7.3 数据访问权限
- 敏感数据仅限产品负责人与数据分析师访问
- 建立数据导出审批流程
