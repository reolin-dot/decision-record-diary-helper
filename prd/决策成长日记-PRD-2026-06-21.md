## 二、产品背景与问题定义

### 2.1 我们为什么要做这个产品？

每个人每天都要做无数个决定——小到「今天吃什么」，大到「要不要跳槽」「要不要和这个人结婚」。

但现实中，大多数年轻人：
- **做了决定就忘了**：不知道自己当初为什么这样选，也无法从结果中学习
- **选择焦虑严重**：面对重要决定时反复纠结，浪费大量情绪能量
- **决策能力没有成长**：同样的错误反复犯，没有建立自己的「决策方法论」
- **缺少复盘习惯**：即便想复盘，也没有合适的工具和框架

### 2.2 产品核心价值

> **把每一次选择，变成你成长的养料。**

决策成长日记不是一个「帮你做决定」的工具，而是一个**帮你记录决策、追踪结果、从结果中学习**的成长伴侣。

## 四、目标用户画像

### 4.1 主要用户群体

**用户 A：职场纠结星人（占比约 40%）**
- 年龄：25–32 岁，职场 3–8 年
- 特征：面临职业选择多（跳槽、转行、晋升），容易反复纠结
- 痛点：做了决定总怀疑自己，想复盘但不知道怎么入手
- 使用场景：跳槽前记录决策 → 3 个月后复盘结果

**用户 B：人生重大决策焦虑者（占比约 35%）**
- 年龄：28–35 岁
- 特征：面临结婚、买房、生娃等人生重大决策
- 痛点：担心选错影响一生，希望能有框架辅助思考
- 使用场景：重大决策前用「锦囊」辅助分析，记录决策过程

**用户 C：自我成长爱好者（占比约 25%）**
- 年龄：22–30 岁，有写日记/复盘习惯
- 特征：关注自我认知，愿意为成长付费
- 痛点：现有日记工具太泛，没有专门针对「决策」的维度
- 使用场景：定期回顾自己的决策模式，生成年度决策报告

### 4.2 用户核心需求优先级

| 需求 | 重要性 | 紧迫性 | 优先级 |
|------|--------|--------|--------|
| 记录决策过程 | 高 | 高 | P0 |
| 辅助决策分析 | 高 | 中 | P0 |
| 追踪决策结果 | 高 | 中 | P0 |
| 了解自己决策风格 | 中 | 中 | P1 |
| 定期复盘提醒 | 中 | 低 | P1 |

### 功能模块 1：决策性格测试（Onboarding）

#### 功能概述
新用户首次打开小程序，通过一个 10 题的情境测试，了解自己的决策风格，并据此获得个性化的产品使用建议。

#### 业务规则
- 测试为**首次使用必做**，但允许用户跳过（跳过则使用默认风格）
- 测试完成后生成「决策风格画像」，永久保存，用户可随时重新测试
- 测试结果不影响功能可用性，只影响**推荐内容和提示语**

#### 决策风格分类

| 风格类型 | 特征 | 推荐功能 |
|---------|------|---------|
| 🔥 冲动型 | 决定快，但容易后悔 | 推荐「24 小时冷却期」功能 |
| 🧊 犹豫型 | 反复分析，难行动 | 推荐「5 分钟限时决策」功能 |
| ⚖️ 理性型 | 依赖数据，忽略直觉 | 推荐「身体感受记录」功能 |
| 💫 直觉型 | 跟随感觉，偶尔翻车 | 推荐「结果追踪强化」功能 |

#### 交互流程

```
进入小程序
    ↓
判断是否已测试
    ├── 已测试 → 直接进入首页
    └── 未测试 → 进入测试流程
                    ↓
          展示测试介绍页（说明测试目的，约 1 分钟）
                    ↓
          逐题展示（共 10 题，每题单选，支持上一题/下一题）
                    ↓
          答题进度条展示
                    ↓
          答完 10 题 → 生成结果页（风格画像 + 个性化建议）
                    ↓
          点击「开始使用」→ 进入首页
```

#### 题目示例（供参考）

1. **朋友约你周末出去玩，但你很累，你会？**
   - A. 直接答应，怕拒绝会尴尬（冲动型倾向）
   - B. 先说考虑一下，然后纠结半天（犹豫型倾向）
   - C. 列出「去的好处/不去的好处」再决定（理性型倾向）
   - D. 感受一下自己内心最想去吗（直觉型倾向）

#### 输出物
- 结果页包含：风格名称 + 风格描述 + 3 条个性化使用建议
- 结果自动保存至用户个人中心，可随时查看

#### 非目标
- 不做「科学级」心理学测评（不需要达到学术标准）
- 不根据测试结果限制功能使用

### 功能模块 2：记录决策（核心流程）

#### 功能概述
用户遇到需要决策的事情时，通过引导式流程快速记录决策内容，设定复盘时间。

#### 业务规则
- 决策记录支持**草稿自动保存**（中途退出可恢复）
- 必填字段：决策标题、最终选择
- 选填字段：选项列表、决策背景、情绪状态
- 复盘时间支持预设选项（1 周后、1 个月后、3 个月后、自定义）

#### 交互流程（4 步引导）

**第 1 步：描述决策**
```
页面标题：记录一个新的决定
输入项：
  - 决策标题（必填，限 50 字，提示语：「比如：要不要接受 A 公司的 offer」）
  - 决策背景（选填，限 500 字，提示语：「发生了什么？为什么现在需要做这个决定？」）
  - 当前情绪（选填，心情标签选择：😰 焦虑 / 🤔 纠结 / 😤 冲动 / 😌 平静 / 其他）
操作按钮：
  - 「下一步」（标题为空时置灰）
  - 「跳过，直接记录」（右上角，高级用户快速通道）
```

**第 2 步：列出选项**
```
页面标题：你有哪些选择？
输入项：
  - 选项 A（必填，提示语：「比如：接受 A 公司 offer」）
  - 选项 B（必填）
  - 添加选项（按钮，最多支持 6 个选项）
  - 每个选项可展开「添加备注」（选填，记录对该选项的顾虑/期待）
操作按钮：「下一步」
特殊逻辑：
  - 如果用户在第一步选择了「快速记录」，此步骤可跳过
  - 选项少于 2 个时，提示「至少有 2 个选项才能做决策哦」
```

**第 3 步：做出选择**
```
页面标题：你最终选择了？
交互方式：
  - 展示第 2 步录入的所有选项（卡片式展示）
  - 用户点击选择其中一个（单选）
  - 选中后该卡片高亮，显示「确定选这个吗？」确认弹窗
输入项：
  - 选择理由（选填，限 200 字，提示语：「记下你为什么这样选，以后复盘会很有价值」）
  - 期望结果（选填，限 200 字，提示语：「你希望这件事的结果是什么样的？」）
操作按钮：「确定选择」
```

**第 4 步：设定复盘时间**
```
页面标题：什么时候回顾这个结果？
交互方式：
  - 快捷选项按钮：
    - 「1 周后」（+7 天）
    - 「1 个月后」（+30 天）
    - 「3 个月后」（+90 天）
  - 自定义日期选择器（日期控件）
  - 也可以「暂时不设定」（后续手动设定）
操作按钮：「完成，种下这颗种子 🌱」
完成反馈：
  - 展示成功动画（种子入土动画）
  - 提示语：「已记录！到复盘日我会提醒你 🌱」
  - 按钮：「返回首页」
```

#### 数据字段定义

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| decision_id | String | 是 | 唯一标识 |
| title | String | 是 | 决策标题 |
| background | String | 否 | 决策背景 |
| mood_tag | String | 否 | 情绪标签 |
| options | Array | 是 | 选项列表（含备注） |
| chosen_option | String | 是 | 最终选择 |
| reason | String | 否 | 选择理由 |
| expected_result | String | 否 | 期望结果 |
| review_date | Date | 否 | 复盘日期 |
| status | Enum | 是 | 待复盘/已复盘/已取消 |
| created_at | Date | 是 | 创建时间 |
| updated_at | Date | 是 | 更新时间 |

#### 非目标
- 不支持多人协作决策
- 不支持图片/文件附件上传（MVP 阶段）

### 功能模块 3：决策锦囊（辅助决策）

#### 功能概述
用户在纠结某个决策时，通过「锦囊」获得结构化的决策辅助框架和引导式分析，帮助用户理清思路。

#### 业务规则
- 锦囊**不替代用户做决定**，只提供分析框架和引导提问
- 锦囊内容基于用户选择的「困境类型」匹配
- MVP 版本内置 5 种经典决策框架，不接入 AI

#### 困境类型与框架匹配

| 困境类型 | 适用场景 | 推荐框架 |
|---------|---------|---------|
| 🤔 二选一困难 | 两个选项各有利弊，无法取舍 | 利弊清单 + 10-10-10 法则 |
| 💼 职业选择 | 跳槽、转行、晋升等职业决策 | SWOT 分析 |
| ❤️ 人际关系 | 表白、分手、朋友矛盾等 | 核心价值观对齐法 |
| 💰 财务决策 | 买东西、投资、消费等 | 机会成本分析 |
| 🌱 长期规划 | 人生方向、目标设定等 | 价值观排序法 |

#### 交互流程

```
首页 → 点击「决策锦囊」入口
    ↓
选择困境类型（5 个类型卡片，带图标和说明）
    ↓
系统推荐匹配框架（展示框架说明，用户可切换其他框架）
    ↓
进入引导式分析（按框架逐步引导）
    ↓
生成「决策参考卡片」（汇总分析结果）
    ↓
用户可「保存到我的决策」（自动创建决策记录草稿）
```

#### 各框架引导内容（详细）

**框架 1：利弊清单（适用于二选一）**
```
引导步骤：
  第 1 步：列出选项 A 的所有好处（至少 3 条）
  第 2 步：列出选项 A 的所有坏处（至少 3 条）
  第 3 步：列出选项 B 的所有好处（至少 3 条）
  第 4 步：列出选项 B 的所有坏处（至少 3 条）
  第 5 步：给每个选项的「重要性」打分（1-5 分）
输出：利弊对比表 + 加权得分参考
```

**框架 2：10-10-10 法则（适用于大多数决策）**
```
引导步骤：
  提问 1：10 分钟后，这个决定让我感觉如何？
  提问 2：10 个月后，如果结果好，我的生活有什么变化？如果结果不好呢？
  提问 3：10 年后，回望今天，这个决定对我想成为的人有什么影响？
输出：三个时间维度的思考汇总
```

**框架 3：SWOT 分析（适用于职业选择）**
```
引导步骤：
  S（优势）：选择这个选项，我的优势是什么？
  W（劣势）：选择这个选项，我的劣势是什么？
  O（机会）：这个选项能带来什么机会？
  T（威胁）：这个选项有什么风险？
输出：SWOT 矩阵图
```

#### 非目标
- MVP 不接入 AI 生成内容（避免幻觉风险）
- 不保存锦囊使用记录（V1.0 加入）

### 功能模块 4：复盘提醒与追踪

#### 功能概述
在用户设定的复盘时间到达时，通过微信服务通知推送提醒，引导用户完成决策复盘，记录实际结果。

#### 业务规则
- 复盘提醒通过**微信订阅消息**发送（需用户授权）
- 用户可随时手动复盘（不依赖推送）
- 复盘问卷设计为**轻量级**（3 个问题，30 秒完成），降低完成门槛
- 复盘完成后，决策状态从「待复盘」变为「已复盘」

#### 复盘问卷设计

| 问题 | 类型 | 必答 | 说明 |
|------|------|------|------|
| 结果如何？ | 单选 | 是 | 超出预期 / 符合预期 / 一般 / 不如预期 / 非常糟糕 |
| 实际结果和你的期望有什么不同？ | 文本题 | 否 | 限 200 字，提示语：「有什么出乎意料的吗？」 |
| 下次遇到类似情况，你会怎么做？ | 文本题 | 否 | 限 200 字，提示语：「你学到了什么？」 |

#### 推送策略

| 时机 | 内容 | 跳转 |
|------|------|------|
| 复盘日当天早上 9:00 | 「你 7 天前种下的种子该浇水啦 🌱」 | 复盘问卷页 |
| 复盘日未操作，3 天后 | 「还记得你做的那个决定吗？花 30 秒回顾一下吧」 | 复盘问卷页 |
| 复盘日未操作，7 天后 | 「再不回顾，你可能就忘了这个决定了哦」 | 复盘问卷页（弱化推送频率） |

#### 用户控制
- 用户可在「设置」中关闭复盘提醒
- 用户可修改复盘时间（在决策详情页）
- 用户可删除决策记录（软删除，数据保留用于匿名统计）

#### 非目标
- 不做复杂的推送策略算法（MVP 用固定规则）
- 不强制用户复盘（可跳过）

### 功能模块 5：决策花园可视化（V1.0）

#### 功能概述
将用户的决策记录以「花园」形式可视化展示，每个决策是一棵植物，状态随复盘结果变化，增强产品趣味性和长期使用动力。

#### 植物状态定义

| 状态 | 触发条件 | 视觉表现 |
|------|---------|---------|
| 🌱 种子 | 决策已记录，未到复盘日 | 土地里有一颗种子 |
| 🌿 发芽 | 已到复盘日，但未复盘 | 种子冒出绿色嫩芽 |
| 🌸 开花 | 复盘结果：超出预期/符合预期 | 植物开花，颜色鲜艳 |
| 🍂 枯萎 | 复盘结果：不如预期/非常糟糕 | 植物枯萎，但保留（表示「这也是养分」） |
| 🌳 大树 | 同一类型决策连续 3 次结果好 | 植物长成大树（成就系统） |

#### 交互设计
- 花园视图按时间顺序排列植物
- 点击植物可查看决策详情和复盘记录
- 花园支持「只看开花」「只看枯萎」等筛选
- 首次开花时有动画庆祝效果

#### 非目标
- MVP 不做 3D 花园（用 2D 图标即可）
- 不与其他用户花园互动（无社交）

### 功能模块 6：个人中心与数据统计

#### 功能概述
展示用户的决策风格、决策数据统计、成长轨迹。

#### 数据指标

| 指标 | 说明 |
|------|------|
| 总决策数 | 已记录的决策总数 |
| 复盘完成率 | 已复盘数 / 应设置复盘数 |
| 决策风格分布 | 各类决策（职业/人际/财务等）占比 |
| 决策正确率 | 结果为「符合/超出预期」的占比 |
| 连续记录天数 | Streak 机制，增强留存 |
| 成就徽章 | 完成特定行为解锁（如：连续记录 7 天） |

#### 决策年度报告（V1.0 后期）
- 每年 12 月底自动生成
- 包含：年度决策数量、决策风格变化、最大收获、成长轨迹
- 支持分享到微信好友/朋友圈（图片形式）

## 九、成功指标（如何衡量产品是否成功）

### 9.1 北极星指标

> **每月完成有效复盘的决策数**（有效复盘 = 用户回答了至少 1 个复盘问题）

**为什么选这个？**
- 直接反映产品核心价值（帮助用户从决策中学习）
- 比「注册用户数」更能反映真实产品价值
- 可驱动产品迭代方向

### 9.2 关键指标层级

```
北极星指标：每月完成有效复盘的决策数
    │
    ├── 驱动指标（影响北极星的直接因素）
    │   ├── 每周新增决策记录数
    │   ├── 复盘完成率（目标 > 60%）
    │   └── 用户活跃天数/月（目标 > 4 天）
    │
    └── 健康指标（反映产品健康度）
        ├── 次日留存率（目标 > 30%）
        ├── 7 日留存率（目标 > 15%）
        ├── 30 日留存率（目标 > 10%）
        └── NPS 净推荐值（目标 > 30）
```

### 9.3 MVP 阶段目标（前 3 个月）

| 指标 | 目标值 | 衡量方式 |
|------|--------|---------|
| 注册用户数 | 1000+ | 微信小程序后台 |
| 人均记录决策数 | > 3 个/月 | 数据库统计 |
| 复盘完成率 | > 50% | 数据库统计 |
| 次日留存 | > 25% | 微信小程序数据助手 |
| 用户反馈满意度 | > 4.0/5.0 | 小程序内评分 |

## 十二、风险与缓解措施

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|---------|
| 用户不愿意复盘（复盘完成率低） | 高 | 高 | 复盘问卷设计极简（3 个问题）；加入游戏化元素（成就、Streak） |
| 微信订阅消息审核不通过 | 中 | 中 | 提前申请，准备多个模板方案；同时支持手动复盘 |
| 用户量增长后云开发费用超预算 | 低 | 中 | MVP 阶段用户量小，暂不需担心；V1.0 引入 BaaS 方案评估 |
| 决策框架内容质量不足 | 中 | 中 | 邀请心理咨询师/决策专家审核内容；V1.0 接入 AI 辅助 |
| 用户混淆「锦囊」和「帮我做决定」 | 中 | 低 | 产品内明确文案说明；锦囊结果页强调「这是参考，最终决定权在你」 |

## 十三、附录：用户故事完整列表

### 核心用户故事

**作为新用户，我希望…**
1. 首次打开小程序时能了解这个产品是做什么的，以便决定是否继续使用 → *决策性格测试介绍页*
2. 通过一个有趣的测试了解自己的决策风格，以便更好地使用产品 → *决策性格测试*

**作为正在纠结的用户，我希望…**
3. 遇到难以决定的事情时，能快速记录我的决策过程，以便后续回顾 → *记录决策流程*
4. 在记录决策时获得一些分析框架的帮助，以便更全面地思考 → *决策锦囊*
5. 能看到我之前做过的类似决策和结果，以便参考 → *决策搜索/筛选（P1）*

**作为关注成长的用户，我希望…**
6. 在决策结果出来后能收到提醒，以便及时复盘 → *复盘提醒*
7. 复盘时能有引导性的问题，以便更有针对性地思考 → *复盘问卷*
8. 能看到我一段时间内的决策模式和成长轨迹，以便了解自己 → *个人中心数据统计*
9. 能看到我的决策以有趣的方式展示（花园），以便增加使用动力 → *决策花园*

**作为忙碌的用户，我希望…**
10. 能快速记录决策（不用填太多信息），以便降低使用门槛 → *快速记录模式*
11. 能修改复盘提醒时间，以便匹配我的实际情况 → *修改复盘时间*
